Different Approaches, Different Guarantees

Each tool uses a fundamentally different approach to finding bugs. Understanding the approach explains the guarantees.

Semgrep

Pattern Matching

Searches code for patterns that look like known bug types. Fast and easy to write rules for, but fundamentally limited: it can only find bugs that match a pattern someone already wrote.

Strengths: Speed, ease of custom rules, large community, 30+ languages. Limitation: Can’t reason about values, paths, or program state.

CodeQL

Semantic Queries

Builds a database of your code’s structure and lets you query it with a SQL-like language. More powerful than pattern matching because it understands data flow, but still relies on human-written queries.

Strengths: Powerful query language, data flow analysis, GitHub integration. Limitation: No formal proofs, results depend on query quality.

SonarQube

Rule Engine

Applies a large set of predefined rules covering code smells, bugs, and vulnerabilities. Excellent for code quality metrics and team dashboards, but rules are heuristic — they report what might be wrong.

Strengths: Enterprise dashboard, quality gates, large rule set, many languages. Limitation: High false positive rate, no formal guarantees.

Facebook Infer

Abstract Interpretation

Uses separation logic and abstract interpretation to find memory bugs and concurrency errors. The closest tool to AEON in philosophy — but limited to 4 languages and a few analysis types.

Strengths: Sound analysis, separation logic, used at Facebook scale. Limitation: 4 languages only, no contracts, no taint, no termination.

Why AEON Is Different

AEON doesn’t replace these tools — it operates at a fundamentally different level.

AEON

30+ Formal Methods

AEON combines 30+ peer-reviewed formal verification engines into a single tool. Each engine catches a different class of bugs: null dereferences, integer overflow, SQL injection, memory leaks, dead code, race conditions, API misuse, swallowed exceptions, and more.

The key difference: when AEON says your code is correct, it’s not an opinion or a heuristic — it’s a mathematical proof. When it finds a bug, it gives you the exact input that triggers it.

Unique strengths: Mathematical proofs, 44 peer-reviewed papers, all-path coverage, contract verification, termination proofs, auto-fix with confidence scores, plain-English explanations. Current limitation: 14 languages (vs. 30+ for pattern matchers).
