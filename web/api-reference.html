API Reference

REST API endpoints and Python SDK for programmatic verification.

REST API

Start the AEON API server to verify code programmatically over HTTP. All endpoints accept and return JSON.

# Start the API server
python -m aeon.api_server

# Server runs at http://localhost:5000
# Health check: GET /health

Authentication

The local API server runs without authentication by default. For production deployments, set the AEON_API_KEY environment variable. When set, all requests must include the header:

Authorization: Bearer <your-api-key>
POST /verify

Verify a source code string. Returns verification results including any bugs found, engine details, and suggested fixes.

Request Body

ParameterTypeRequiredDescription
codestringrequiredSource code to verify
languagestringoptionalLanguage override (auto-detected if omitted)
deep_verifybooleanoptionalEnable all 30+ engines (default: false)
enginesstring[]optionalSpecific engines to run: ["symbolic", "hoare", "taint"]
timeoutintegeroptionalTimeout in seconds (default: 30)

Example Request

curl -X POST http://localhost:5000/verify \
  -H "Content-Type: application/json" \
  -d '{
    "code": "def divide(a, b):\n    return a / b",
    "language": "python",
    "deep_verify": true
  }'

POST /verify/file

Verify a file by path. The file must be accessible from the server's filesystem.

ParameterTypeRequiredDescription
file_pathstringrequiredAbsolute or relative path to source file
deep_verifybooleanoptionalEnable all 30+ engines (default: false)
enginesstring[]optionalSpecific engines to run

Example

curl -X POST http://localhost:5000/verify/file \
  -H "Content-Type: application/json" \
  -d '{"file_path": "src/payment.py", "deep_verify": true}'
POST /verify/directory

Scan an entire directory recursively. Equivalent to aeon scan via the API.

ParameterTypeRequiredDescription
directorystringrequiredPath to directory to scan
deep_verifybooleanoptionalEnable all engines
parallelbooleanoptionalEnable parallel scanning
workersintegeroptionalNumber of parallel workers
baselinestringoptionalPath to baseline file for diff mode
formatstringoptionaljson | sarif

Python SDK

The Python SDK provides a programmatic interface to AEON's verification engines. Install with pip install aeon-lang.

from aeon import verify, scan, AeonConfig

verify(code, language=None, deep_verify=False, engines=None, timeout=30)

Verify a source code string. Returns a VerificationResult.

ParameterTypeDefaultDescription
codestr—Source code to verify
languagestr | NoneNoneLanguage (auto-detected if None)
deep_verifyboolFalseEnable all 30+ engines
engineslist[str] | NoneNoneSpecific engines to run
timeoutint30Timeout in seconds
 
from aeon import verify

result = verify("""
def average(nums: list, count: int) -> float:
    \"\"\"
    Requires: count != 0
    \"\"\"
    return sum(nums) / count
""", language="python", deep_verify=True)

print(result.verified)           # True
print(result.bugs_found)         # 0
print(result.functions_analyzed) # 1
print(result.execution_time_ms)  # 420

verify_file(path, deep_verify=False, engines=None, timeout=30)

Verify a file by path. Language is auto-detected from extension.

from aeon import verify_file

result = verify_file("src/payment.py", deep_verify=True)
if not result.verified:
    for bug in result.results:
        print(f"[{bug.code}] {bug.message} at line {bug.line}")
        print(f"  Engine: {bug.engine}")
        print(f"  Fix: {bug.fix}")

scan(directory, deep_verify=False, parallel=False, workers=None, baseline=None)

Scan a directory recursively. Returns a ScanResult with per-file results.

from aeon import scan

results = scan("src/", deep_verify=True, parallel=True, workers=8)

print(f"Files scanned: {results.files_scanned}")
print(f"Total bugs: {results.total_bugs}")
print(f"Clean files: {results.clean_files}")

for file_result in results.file_results:
    if not file_result.verified:
        print(f"\n{file_result.file_path}:")
        for bug in file_result.results:
            print(f"  [{bug.code}] {bug.message}")

AeonConfig(config_path=None, **overrides)

Load and customize AEON configuration programmatically.

from aeon import AeonConfig, scan

config = AeonConfig(
    deep_verify=True,
    parallel=True,
    workers=8,
    timeout=60,
    exclude=["**/test_*.py", "**/vendor/**"],
    security={
        "auto_label": True,
        "patterns": {
            "SECRET": ["*_key", "*_token", "password*"]
        }
    }
)

results = scan("src/", config=config)

Return Types

VerificationResult

FieldTypeDescription
verifiedboolTrue if no bugs found
bugs_foundintNumber of bugs found
functions_analyzedintNumber of functions analyzed
classes_analyzedintNumber of classes analyzed
resultslist[Finding]List of findings (bugs/warnings)
execution_time_msintTotal execution time in milliseconds
engines_usedlist[str]Engines that were run

Finding

FieldTypeDescription
typestr"error" or "warning"
codestrError code (e.g., "AEON-E001")
messagestrHuman-readable description
enginestrEngine that found this issue
lineintLine number
columnintColumn number
severitystr"error" | "warning" | "info"
fixstr | NoneSuggested fix
detailsdictEngine-specific details (counterexamples, ranges, etc.)

FieldTypeDescription
