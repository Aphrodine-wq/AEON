AEON Native Language

Built-in language examples

The AEON native language uses three primitives: pure, task, and data.

Pure Functions

Side-effect-free functions with contracts verified at compile time.

pure add(a: Int, b: Int) -> Int {
  ensures: result == a + b
  return a + b
}

pure abs(x: Int) -> Int {
  ensures: result >= 0
  if x >= 0 { return x }
  else { return 0 - x }
}

Contract System

Preconditions and postconditions verified by Z3 and Hoare logic.

pure processPayment(amount: Int, balance: Int) -> Int {
  requires: balance >= amount
  requires: amount > 0
  ensures:  result == balance - amount
  return balance - amount
}

pure clamp(value: Int, lo: Int, hi: Int) -> Int {
  requires: lo <= hi
  ensures:  result >= lo
  ensures:  result <= hi
  ...
}

Task Functions

Functions with declared algebraic effects tracked by the compiler.

task createUser(user: User) -> Bool {
  requires: user.email.isValid()
  ensures:  result == true
  effects:  [Database.Write]
  return db.insert(user)
}

task findUser(id: UUID) -> User {
  effects: [Database.Read]
  return db.find(id)
}

Termination Proofs

AEON proves recursive functions terminate using ranking functions, size-change analysis, and Houdini invariant inference.

pure factorial(n: Int) -> Int {
  requires: n >= 0
  ensures:  result >= 1
  -- AEON: ranking fn r = n, decreases by 1 each call
  -- Base case: n <= 1 returns without recursion ✓
  if n <= 1 { return 1 }
  return n * factorial(n - 1)
}

pure gcd(a: Int, b: Int) -> Int {
  requires: a > 0 && b > 0
  ensures:  result > 0
  -- AEON: ranking fn r = b, a % b < b ✓
  if b == 0 { return a }
  return gcd(b, a % b)
}
